1 /* mwm 11.05.2011
2  *
3  * An object which does serial communication with the hexacopter. The
4  * idea being that one can simply put this into other code, and use it
5  * there (e.g. in a simulator to read the RC channels, or for HILS).
6  *
7  * --HOW TO USE:--------------------------------------------------------
8  * > To use this in a separate project, you must include this header file,
9  * i.e.
10  * #include "PATH_TO_FILES/serialCommunicator.h"
11  *
12  * > You should include the project "SerialComms.vcproj" in your project
13  *
14  * > Set the dependencies: under "Project Dependecies" check "Serial Communication"
15  * for your project.
16  *
17  * > In your code, instantiate the object with something like
18  * SerialCommunicator* communicator = new SerialCommunicator();
19  * You can test whether a port was opened using by evaluating
20  * communicator->portSuccessfullyOpened()
21  * which returns zero if it failed.
22  *
23  * > The SerialCommunicator runs on multiple threads, which you must create.
24  * This is always done by using the following syntax:
25  * CreateThread(0,0,THREAD_NAME,POINTER_TO_SC,0,0);
26  * where THREAD_NAME is the name of the function to be used, one of the
27  * functions in the namespace "communicationThread". POINTER_TO_SC is the
28  * pointer to the serialCommunicator you have instantiated.
29  * See the function declarations for details on what the threads do.
30  * Note that the threads use various times to Sleep(.), and that the user
31  * can overwrite these files using the appropriate setXX(.) functions. This
32  * must be done *BEFORE* the threads are initialised, otherwise it will have
33  * no effect. The threads are defined in "threadFunction.cpp"
34  *
35  * > In your own code, note the difference between "requestXX" functions,
36  * and "requestConfirmedXX": requestXX only adds a data packet to the queue
37  * and then immediately exits. The command is sent by the transmission thread.
38  * RequestConfirmedXX does not exit until a command has been completed, and
39  * reports success (true) or failure (false). It fails at timeout (i.e. if it
40  * doesn't succeed in some time, it returns false).
41  *
42  * > The "sendXXRequestNow()" functions are intended for use *only* by
43  * the appropriate threads.
44  *
45  */
